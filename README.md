# Krax Project Auto-Setup Script
## Автоматический скрипт для создания проектов Krax с полной структурой и автоматической загрузкой на GitHub. Скрипт самостоятельно удаляется после использования.

### Что делает скрипт
### - Создает полную структуру проекта Krax

### - Настраивает VS Code для отладки

### - Клонирует зависимости (pyplc, pysca)

### - Создает репозиторий на GitHub

### - Пушит код автоматически

### - САМОУДАЛЯЕТСЯ после выполнения (опционально)

## Предварительная настройка
## 1. Установите Git
### bash
### sudo apt update
### sudo apt install git
## 2. Установите GitHub CLI
### bash
### sudo apt update
### sudo apt install gh

## 3. Авторизуйтесь в GitHub CLI
### bash
### gh auth login
## Следуйте инструкциям:

### Выберите GitHub.com

### Выберите HTTPS

### Выберите Login with a web browser

### Нажмите Enter, откроется браузер

### Авторизуйтесь в GitHub

### Вернитесь в терминал

## Проверьте авторизацию:

### bash
### gh auth status
## Использование скрипта
## 1. Клонируйте репозиторий со скриптом:
### bash
### git clone https://github.com/chkrain/setup-project-krax.git
### cd setup-project-krax
## 2. Запустите скрипт:
### bash
### chmod +x setup-project-krax.sh
### ./setup-project-krax.sh
## 3. Скрипт спросит параметры:
### Основные параметры:

### Название репозитория - имя вашего проекта на GitHub

### Описание репозитория - краткое описание проекта

### Основная ветка - обычно main

### Видимость репозитория - public (публичный) или private (приватный)

### Клонировать зависимости - y для автоматического клонирования pyplc/pysca

## Новый параметр:

### Удалить этот скрипт после создания проекта? - y для автоудаления

## 4. Если ответите "y" на автоудаление:
### Скрипт создаст новый проект
### Закоммитит все файлы КРОМЕ самого скрипта и README.md
### Удалит директорию со скриптом
### Автоматически перейдет в новую директорию проекта

## 5. Если ответите "n" на автоудаление:
### Скрипт работает как раньше, оставляет себя в проекте
